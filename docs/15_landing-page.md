# 15. ランディングページ（LP）

## 概要
未ログインユーザーにEdgeConnectの価値を伝え、事業主登録・お客さん利用の両方への導線を提供する。

## URL設計

| URL | 表示条件 | 内容 |
|-----|---------|------|
| `/` | 未ログイン | LP表示 |
| `/` | ログイン済み | 既存ダッシュボード（ロール別） |
| `/about` | 常時 | LP（footerからリンク、シェア用固定URL） |

`/` の未ログイン時と `/about` は同じLPコンポーネントを共有する。

## LPコンテンツ構成（上から順）

| セクション | 内容 |
|-----------|------|
| ヒーロー | キャッチコピー + サービス一言説明 + CTA「無料ではじめる」 |
| 課題提起 | 「予約管理、こんな悩みありませんか？」3つの課題カード |
| 解決策 | EdgeConnectで解決できること（LINE連携・カレンダー同期・QR発行等） |
| 使い方ステップ | 事業主: 3ステップ（登録→メニュー設定→QR共有）、お客さん: 3ステップ（QR読取→メニュー選択→予約確定） |
| 料金 | 無料プラン（現状の機能）、有料プラン（Coming Soon） |
| CTA（再掲） | 「無料ではじめる」「事業主を探す → /explore」 |
| Footer | サイト内リンク（EdgeConnectとは・事業主を探す・利用規約・プライバシーポリシー） |

## CTA遷移先
- 「無料ではじめる」→ LINEログイン → 事業主登録フロー
- 「事業主を探す」→ `/explore`
- 「ログイン」→ LINEログイン → ダッシュボード
