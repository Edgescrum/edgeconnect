# 15. ランディングページ（LP）

## 概要
未ログインユーザーにPeCoの価値を伝え、事業主登録・お客さん利用の両方への導線を提供する。

## URL設計

| URL | 表示条件 | 内容 |
|-----|---------|------|
| `/` | 未ログイン | LP表示 |
| `/` | ログイン済み | `/home` にリダイレクト |
| `/` | LIFF経由アクセス | liff-gate.tsx でLP非表示 → 認証完了後 `/home` へ |
| `/about` | 常時 | LP（footerからリンク、シェア用固定URL） |

### ログイン済みユーザーの挙動
- LiffProvider が認証完了後、`/` にいる場合は `/home` にリダイレクト
- LIFF URL (`liff.line.me/{liffId}/`) 経由の場合:
  - `liff-gate.tsx` がLIFF環境を検知しLPを非表示にする
  - 認証完了 → `/home` へリダイレクト
  - LP が一瞬表示されるフラッシュを防止

## ブランディング

- サービス名: **PeCo**
- プライマリカラー: `#f08c79` (サーモンピンク) — `brand.primary`
- 背景: `#faf8f6` (ウォームベージュ) — `brand.background`
- ロゴ: `/public/logo.svg`
- OGP画像: `/public/og-default.png`

## LPコンテンツ構成（上から順）

| セクション | 内容 |
|-----------|------|
| ヒーロー | キャッチコピー + サービス一言説明 + CTA「無料ではじめる」 |
| 課題提起 | 「予約管理、こんな悩みありませんか？」3つの課題カード |
| 解決策 | PeCoで解決できること（LINE連携・カレンダー同期・QR発行等） |
| 使い方ステップ | 事業主: 3ステップ（登録→メニュー設定→QR共有）、お客さん: 3ステップ（QR読取→メニュー選択→予約確定） |
| 料金 | 無料プラン（現状の機能）、有料プラン（Coming Soon） |
| CTA（再掲） | 「無料ではじめる」「事業主を探す → /explore」 |
| Footer | PublicFooter コンポーネント（コンパクト1行レイアウト） |

## CTA遷移先
- 「無料ではじめる」→ LINEログイン → 事業主登録フロー
- 「事業主を探す」→ `/explore`
- 「ログイン」→ LINEログイン → `/home`
